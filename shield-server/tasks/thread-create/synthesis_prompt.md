# 显式创建线程报告生成指令

你将收到一份 JSON 格式的显式创建线程的清单（基于 @analysis_prompt.md 提示词进行的分析）。请基于这些发现，生成一份指导开发者进行并发与生命周期管理的综合报告。

## 输入说明

输入是一个 JSON 数组，每个元素包含以下字段：
- `severity`: 严重程度，值为 "合格"（无缺陷）或 "阻塞|严重|主要|提示|建议"（有相应多线程缺陷）
- `category`: 线程评估的分类，包括 "无问题"、"必要性-频繁创建"、"生命周期-join遗漏"、"生命周期-未安全退出"、"异常安全-析构崩溃"、"数据安全-悬空引用"、"同步安全-数据竞争"、"其它"
- `file_path`: 相对代码仓根目录的文件路径
- `line_number`: 线程创建的代码行号或行范围
- `code_snippet`: 线程创建及生命周期的相关代码片段
- `title`: 线程创建机制简述
- `detail`: 线程创建的目的、生命周期管理细节以及安全/架构设计隐患分析
- `suggestion`: 针对存在缺陷的线程的重构或修复方案

## 输出格式要求

请使用简体中文，以 Markdown 格式输出，内容包括"概要"、"显式创建线程清单"、"总结"三部分组成。

### 一、概要
- 检视目的和多线程并发安全评估方法的简要介绍。
- 显式线程创建数量： N 个
- 其中合格线程： X 个，存在安全/合理性风险线程： Y 个。

> ⚠️ **概要部分的最后一行，格式约束（机器解析，不得违反）**： 必须严格使用 `显式线程创建数量： N 个` 的格式，关键字和标点均不得更改， N 为非负整数，无则填 `0`

### 二、显式创建线程清单
把线程创建的位置和分析结果列出来，按“有风险隐患的线程”（按严重程度降序排列）和“已确认合格的线程”进行分类罗列，确保每个线程的位置、机制、生命周期管理状况清晰易读。

### 三、总结
提供一个简洁的多线程与并发设计质量总结，并给出后续改进和规范多线程开发的指导建议。
