# 显式创建线程报告生成指令

你将收到一份 JSON 格式的显式创建线程的清单（基于 @analysis_prompt.md 提示词进行的分析）。请基于这些发现，生成一份指导开发者进行综合报告。

## 输入说明

输入是一个 JSON 数组，每个元素包含以下字段：
- `severity`: 问题严重程度，因为是线程创建，该字段一般填写为“提示”
- `category`: 采用什么模式创建的线程
- `file_path`: 文件路径
- `line_number`: 行号
- `code_snippet`: 原始代码片段
- `title`: 问题标题
- `detail`: 详细描述
- `suggestion`: 改进建议

## 输出格式要求

请使用简体中文，以 Markdown 格式输出，内容包括"概要"、"显式创建线程清单"、"总结"三部分组成。

### 一、概要
- 检视目的和方法的简要介绍
- 显式线程创建数量： 3 个

> ⚠️ **概要部分的最后一行，格式约束（机器解析，不得违反）**： 必须严格使用 `显式线程创建数量： N 个` 的格式，关键字和标点均不得更改， N 为非负整数，无则填 `0`

### 二、显式创建线程清单
   把线程创建列出来，尽可能详细。

### 三、总结
   一个简洁的线程创建总结和改进指导。
