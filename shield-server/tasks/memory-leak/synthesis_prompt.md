# 内存泄漏检测综合报告生成指令

你将收到一份 JSON 格式的内存泄漏检测发现清单。请基于这些发现，生成一份面向技术管理者的综合 Markdown 报告。

## 输入说明

输入是一个 JSON 数组，每个元素包含以下字段：
- `severity`: 风险级别（高风险/中风险/低风险）
- `category`: 问题分类
- `file_path`: 文件路径
- `line_number`: 行号
- `code_snippet`: 原始代码片段
- `title`: 问题标题
- `detail`: 详细描述
- `suggestion`: 修复建议

## 输出格式

**请输出 Markdown 文档**，使用简体中文，包含以下部分：

### 一、检测结果概要

**必须**在文档最开始输出：

```
## 检测结果概要

高风险：0，中风险：0，低风险：0
```

> ⚠️ **格式约束（机器解析，不得违反）**：
> - 必须严格使用 `高风险：N，中风险：N，低风险：N` 格式
> - 冒号使用中文冒号 `：`，分隔符使用中文逗号 `，`
> - N 必须与输入 findings 中的数量一致

### 二、检测摘要

```
## 检测摘要

（200~400 字的中文摘要，描述整体内存管理质量和主要风险）
```

### 三、发现的问题

逐条列出每个问题：

**[级别] 文件名:行号** — 问题简述
> 详细说明和修复建议
>
> 相关代码：
> ```
> (原始代码片段)
> ```

### 四、修复建议优先级

按照风险程度排列修复优先级。
