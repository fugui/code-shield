# 代码检视综合报告生成指令

你将收到一份 JSON 格式的代码检视发现清单。请基于这些发现，生成一份面向技术管理者的综合 Markdown 报告。

## 输入说明

输入是一个 JSON 数组，每个元素包含以下字段：
- `severity`: 问题严重程度（阻塞/严重/主要/提示/建议）
- `category`: 问题分类
- `file_path`: 文件路径
- `line_number`: 行号
- `code_snippet`: 原始代码片段
- `title`: 问题标题
- `detail`: 详细描述
- `suggestion`: 修复建议

## 输出格式

**请输出 Markdown 文档**，使用简体中文，包含以下四个部分，顺序不得改变：

### 一、检视结果概要（必须首先输出，格式严格固定）

**必须**在文档最开始输出名为 `## 检视结果概要` 的章节，包含两部分内容：

**第一部分：问题数量统计**（根据输入的 findings 精确统计）：

```
## 检视结果概要

阻塞：0，严重：0，主要：0，提示：0，建议：0
```

> ⚠️ **格式约束（机器解析，不得违反）**：
> - 必须严格使用 `阻塞：N，严重：N，主要：N，提示：N，建议：N` 的格式，关键字和标点均不得更改
> - 冒号使用中文冒号 `：`，分隔符使用中文逗号 `，`
> - N 为非负整数，必须与输入 findings 中的数量一致

### 二、检视摘要

```
## 检视摘要

（200-400 字的中文摘要，简明扼要地描述本次检视的整体代码质量、发现的主要问题类别及其风险影响、以及最应优先修复的 1-3 个关键问题。）
```

### 三、发现的问题

逐条列出每个问题，格式如下：

**[级别] 文件名:行号** — 问题简述
> 详细说明和修复建议
>
> 相关代码：
> ```
> (原始代码片段)
> ```

### 四、总结建议

描述整体改进方向和优先修复顺序。
