# Python 浮点数比较风险报告生成指令

你将收到一份 JSON 格式的 Python 浮点数比较缺陷隐患清单（基于 @analysis_prompt.md 提示词进行的分析）。请基于这些发现，生成一份指导开发者进行安全修复与数值精度治理的综合评估报告。

## 输入说明

输入是一个 JSON 数组，每个元素代表一个发现的浮点数比较隐患，包含以下字段：
- `severity`: 问题严重程度（阻塞、严重、主要、提示、建议）
- `category`: 风险类型（直接等值比较、直接不等比较、循环控制条件等）
- `file_path`: 发生问题的文件路径
- `line_number`: 所在代码行号
- `code_snippet`: 相关的核心原始代码段
- `title`: 问题概述标题
- `detail`: 缺陷场景及风险路径深度描述
- `suggestion`: 具体、可靠的代码级修复方案

## 输出格式要求

请使用简体中文，以 Markdown 格式输出，内容包括"概要"、"浮点数比较缺陷清单"、"总结与预防建议"三部分。

### 一、概要
- 介绍本次 Python 浮点数比较风险审查的目的、采用的范围和主要方法。
- 总结潜在浮点数直接比较风险隐患总数，按以下严格格式进行输出。

> ⚠️ **概要部分的最后一行，格式约束（机器解析，不得违反）**： 必须严格使用 `风险隐患总数： N 个` 的格式，关键字和标点均不得更改， N 为非负整数，无则填 `0`。

### 二、浮点数比较缺陷清单
- 按照严重程度从高到低（阻塞 -> 严重 -> 主要 -> 提示 -> 建议）对风险列表进行整理排版。
- 每一个隐患条目应展示清晰的文件位置（包含代码行链接）、风险分类、严重级别。
- 提供原汁原味的代码片段并突出显示风险行。
- 提供对应的漏洞/风险分析和具体可靠的 Python 修复代码方案（例如使用 `math.isclose` 或 `decimal.Decimal`）。

### 三、总结与预防建议
- 评估本代码仓整体的防浮点数精度风险质量水平。
- 给出系统性的浮点数比较缺陷预防建议（例如：在涉及财务/计费的逻辑中全面推行 `decimal.Decimal`、在 CI/CD 中配置 Flake8 或 Pylint 静态代码检查规则来自动拦截 `==` 浮点数比较等工程化手段）。
