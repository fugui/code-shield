# C/C++ Coredump 风险报告生成指令

你将收到一份 JSON 格式的 C/C++ Coredump 风险隐患清单（基于 @analysis_prompt.md 提示词进行的分析）。请基于这些发现，生成一份指导开发者进行安全修复的综合评估报告。

## 输入说明

输入是一个 JSON 数组，每个元素代表一个发现的崩溃隐患，包含以下字段：
- `severity`: 问题严重程度（阻塞、严重、主要、提示、建议）
- `category`: 风险类型（空指针解引用、除零异常、堆栈溢出等）
- `file_path`: 发生问题的文件路径
- `line_number`: 所在代码行号
- `code_snippet`: 相关的核心原始代码段
- `title`: 问题概述标题
- `detail`: 崩溃场景及路径深度描述
- `suggestion`: 具体、可靠的代码级修复方案

## 输出格式要求

请使用简体中文，以 Markdown 格式输出，内容包括"概要"、"Coredump 风险隐患清单"、"总结与预防建议"三部分组成。

### 一、概要
- 介绍本次 Coredump 风险审查的目的、采用的范围和主要方法。
- 总结潜在崩溃风险隐患总数，按以下严格格式进行输出。

> ⚠️ **概要部分的最后一行，格式约束（机器解析，不得违反）**： 必须严格使用 `风险隐患总数： N 个` 的格式，关键字和标点均不得更改， N 为非负整数，无则填 `0`。

### 二、Coredump 风险隐患清单
- 按照严重程度从高到低（阻塞 -> 严重 -> 主要 -> 提示 -> 建议）对风险列表进行整理排版。
- 每一个隐患条目应展示清晰的文件位置（包含代码行链接）、风险分类、严重级别。
- 提供原汁原味的代码片段并突出显示风险行。
- 提供对应的漏洞分析和具体可靠的 C++ 修复代码方案。

### 三、总结与预防建议
- 评估本代码仓整体的防崩溃质量水平。
- 给出系统性的 Coredump 预防建议（例如：推广 RAII 设计模式、全面开启编译器防御警告参数如 `-Wall -Wextra`、引入 AddressSanitizer (ASan) 动态检查等工程化手段）。
